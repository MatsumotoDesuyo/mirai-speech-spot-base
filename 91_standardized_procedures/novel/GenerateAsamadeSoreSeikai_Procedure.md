# Procedure: Generate Asamade Sore Seikai (朝までそれ正解・お題生成)

## 1. 目的 (Purpose)
プロジェクト「サンイチ」のチームメンバー間において、キャラクターや世界観に対する**認識のすり合わせ（Alignment）**と**アイデアの創発（Ideation）**を促進する。
「正解のない問い」への議論を通じて、仕様書にはないキャラクターの「厚み」や「人間味」、世界観の「リアリティ」を言語化し、共有財産とする。

## 2. トリガー (Trigger)
* チームビルディングやアイスブレイクを行う際。
* キャラクターの行動原理や性格設定において、メンバー間で解釈のズレを感じた際。
* シナリオ執筆や設定作成において、アイデアが煮詰まり、新しい視点が欲しい際。

## 3. 入力データ (Input Data)
* `context.novel.md` (特にキャラクターのTips、プロフィール、既出の行動)
* `context.gameExperienceOrchestrator.md` (プロジェクトのトーン＆マナー)

## 4. ステップ (Steps)

### ステップ 1: 「確定事実（Fact）」の抽出と「空白（Void）」の発見
まず、対象とするキャラクターや組織について、資料から**「確定している事実」**をピックアップする。次に、その事実の裏側にある**「まだ語られていない空白」**を探す。

**例：**

* **Fact (事実):** 天城岳人は元マタギであり、都会での生活に順応している。
* **Void (空白):** 順応しているとはいえ、本能レベルで拒否反応を示す瞬間はないか？（ストレスの源泉は？）

### ステップ 2: 「問い（Question）」のジャンプ
発見した「空白」に対し、議論を誘発する具体的な問いを作成する。この際、以下の3つの方向性のいずれかに振る。

* **方向性A（ネタ / 日常）:** * ほっこりする、笑える、可愛げ、生活感を探る。
    * 狙い: キャラクターの「人間味」や「愛嬌」を深める。
* **方向性B（ガチ / 倫理）:** * 倫理観、過去のトラウマ、譲れないルールを探る。
    * 狙い: キャラクターの「行動原理（OS）」や「シリアスな深み」を定義する。
* **方向性C（比喩 / IF）:** * **「もし〇〇だったら？」「〇〇に例えると？」** というアナロジーを用いる。
    * 動物、RPGの職業、家電、文房具、属性など、既存の枠組みに当てはめる。
    * 狙い: 言語化しにくい「雰囲気」や「立ち位置」の認識ズレを直感的に修正する。

### ステップ 3: クオリティ・チェック (Quality Control)
作成した問いが以下の基準を満たしているか確認する。

1.  **「決めつけ」がないか:** 「〇〇な癖があるが～」のように、未確定な設定を前提にしていないか？（×：潔癖症の天城が～ → 〇：マタギの天城が許せない不潔さとは～）
2.  **解釈の余地があるか:** 「はい/いいえ」で終わらず、具体的な名詞やシチュエーションを答える形式になっているか。
3.  **サンイチらしさ:** 物語の文脈（ハードボイルド、SF、サスペンス）やキャラの重要度に即しているか。

### ステップ 4: リスト化と提示 (Listing)
複数のカテゴリ（A, B, C）を混ぜ合わせ、**5～6個の候補**として提示する。
特に、物語の核となるキャラクター（桐生、清花など）の含有率を高める。

* 形式： `【対象キャラクター / カテゴリ】 質問文 + (狙い/意図)`

## 5. 思考プロセス例 (Example)

**例：対象キャラクター「篠原清花」**

1.  **Fact:** 17歳の天才科学者。祖父以外の家族を失っている。ハッカー気質。
2.  **Void:** 「天才」ではない、ただの「17歳」としての顔はどこにある？周囲はどう見ている？
3.  **Questioning:**
    * (ネタ): 彼女がこっそり保存している「年相応の」データとは？
    * (ガチ): 彼女が研究を捨ててでも「守りたい」と思う、論理を超えた存在とは？
    * (比喩): 彼女を「文房具」に例えると何か？（カッターナイフ？それとも多機能ボールペン？）
4.  **Refinement:**
    * Q. 「篠原清花が、もしファンタジーRPGの世界に転生したとしたら、彼女が就く『職業』と、最初に覚える『魔法』は何か？」

## 6. 出力 (Output)
ユーザー（GEXO操作者）が選択可能な、複数の「お題候補リスト」。
