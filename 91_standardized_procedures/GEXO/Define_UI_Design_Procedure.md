# Procedure: Define UI Design with Creative AI

## 1. 手順の目的 (Purpose)
GEXOが持つ抽象的な「体験の核（魂）」や「ビジョン」を、AIパートナーとの対話を通じて言語化し、さらに外部のクリエイティブAI（Figma, Midjourney等）を活用して検証することで、具体的かつ実装可能な「ビジュアル要件定義書」を効率的に作成する。

## 2. 手順のトリガー (Trigger)
UI画面、重要な演出、または特殊な背景など、視覚的な定義が必要なアセットの制作に着手する際。

## 3. ステップ (Steps)

### ステップ1: 体験の核とメタファーの抽出
* **アクション**: GEXOは、そのビジュアルを通じてプレイヤーに提供したい「感情」や「体験」をAIに伝える。
* **AIの役割**: 具体的なデザイン（色や形）の話に入る前に、「それはどのような体験を作らなければいけないか？」という問いを投げかけ、**コンセプトの言語化**を支援する。

### ステップ2: 既存コンテクストとの整合性確認
* **アクション**: AIは、抽出されたコンセプトを、既存の `VisualArchitecture.md` や `ColorCode.md` と照らし合わせる。
* **検証**: 「世界観から逸脱していないか？」「カラーバランスは適切か？」を検証し、修正案を提示する。

### ステップ3: 外部AIへの指示書（プロンプト）作成
* **アクション**: AIは、ステップ1と2で合意した内容を、外部クリエイティブAI（Figma Make等）が理解しやすい構造化された**「日本語のプロンプト」**に変換する。

### ステップ4: プロトタイピングとフィードバック
* **アクション**: GEXOは、AIが作成したプロンプトを外部ツールに入力し、生成結果を確認する。
* **ループ**: 生成物に対し「魂に合致している点」と「違和感がある点」をフィードバックする。AIはこのフィードバックを基にプロンプトを修正し、再生成を促す。この工程を納得いくまで繰り返す。

### ステップ5: 仕様の文書化と格納
* **アクション**: 最終的に合意されたビジュアルイメージと言語定義を統合し、AIが「要件定義書（Markdown）」を作成する。
* **成果物**: 生成AIでは再現しきれなかった「細部のこだわり（アニメーションや質感など）」も、補足事項として言語化して記述する。
