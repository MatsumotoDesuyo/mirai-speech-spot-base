# 標準手順: DefineBackgroundRequirements_Procedure (背景要件定義手順)
# Last Updated: 2025-11-08

## 1. 目的 (Purpose)
GEXOの「決断」の認知的負荷を最小化しつつ、ゲームの世界観に合致する背景アセットの「必須要件」と「ロケハン指針」を迅速に定義する。

## 2. トリガー (Trigger)
アセットリスト（`list.md`）に記載された背景の制作に着手する際。

## 3. ステップ (Steps)

### ステップ 3.1: AIによる「要件（魂）」の提案
AIパートナーは、当該アセットが使用されるシナリオ や世界設定 を分析し、シーンの「魂」（雰囲気、デザイン）に関する複数の選択肢を**「視覚的な参考資料の画像リンクや検索リンク」**と共にGEXOに提案する。

### ステップ 3.2: GEXOによる「要件（魂）」の決断
GEXOは、AIが提案した選択肢をレビューし、ビジョンに最も近いものを選択、または調整指示（例：「Aの雰囲気とBのデザインを融合させてほしい」）を出し、最終的な「要件」を**「決断」**する。

### ステップ 3.3: AIによる「構図」の提案
AIパートナーは、ステップ3.2で確定した「要件」に基づき、シーンの目的（状況説明、感情描写など）を達成するための「構図」を、複数の**「視覚的な参考資料の画像リンクや検索リンク」**と共にGEXOに提案する。

### ステップ 3.4: GEXOによる「構図」の決断
GEXOは、提案された構図をレビューし、最も効果的なものを**「決断」**する。

### ステップ 3.5: AIによる「ロケハン候補地」の提案と実行可能性調査
AIパートナーは、確定した「要件」と「構図」を最もよく体現する実在の「ロケハン候補地」を（検索ツール等を活用し）選定する。
同時に、その候補地での**物理ロケハン（立ち入り、撮影）の実行可能性**（制限、許可要否など）を調査し、GEXOに報告する。

### ステップ 3.6: GEXOによる「最終ロケハン指針」の決断
GEXOは、AIの調査報告に基づき、最終的な「ロケハン候補地」と「撮影指針」を**「決断」**する。
* (注: 著作権上の問題が無い場合、GEXOの決断により、物理ロケハンを「デジタル・ロケハン（AIによるイメージボード生成）」で代替することも可能とする)

### ステップ 3.7: AIによる「要件定義書」の最終化
AIパートナーは、GEXOの全ての「決断」を反映した最終的な「背景画像要件定義書（ドラフト）」を生成し、GEXOの承認を得る。

## 4. 成果物 (Artifacts)
* 各背景アセットに対応する「背景画像要件定義書（ドラフト）」
