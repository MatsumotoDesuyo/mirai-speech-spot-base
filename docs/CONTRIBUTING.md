# みらい演説スポットBase への貢献について (Contributing)

本プロジェクトにご関心をお寄せいただき、誠にありがとうございます！
私たちは、「日本の政治参入における『知的資産による障壁』を限りなくゼロに近づける」というミッションを掲げています。このOSSは、その第一歩となるプラットフォームです。

## 1. プロジェクトのビジョンと制約事項
私たちの開発は、大きな社会課題の解決を目指す「全体ミッション」と、それを具現化する「アプリごとのミッション」の2層構造で進められています。
開発に参加する前に、以下のドキュメントをご一読ください。すべての提案や実装は、これらに合致するかどうかで判断されます（ミッションオーナーによるGo/No-Goの判断基準となります）。

### 🌍 プロジェクト全体のビジョン（Why）
プロジェクト全体の根本的な目的と、絶対に守るべきルールです。
* **[01_Mission.md](../01_value_strategy/01_Mission.md)**: プロジェクト全体の存在意義。
* **[02_NorthStar.md](../01_value_strategy/02_NorthStar.md)**: プロジェクト全体が目指す究極の姿。
* **[03_Values_and_Constraints.md](../01_value_strategy/03_Values_and_Constraints.md)**: 遵守すべき法令や行動規範。

### 📱 みらい演説スポットBaseのビジョン（How）
上記の全体ミッションを、このアプリを通じてどのように実現するかの具体論です。
* **[01_Product_Mission.md](../10_speech_spot_app/01_value_strategy/01_Product_Mission.md)**: このアプリが担う具体的な役割と価値創造の定義。
* **[02_Product_NorthStar.md](../10_speech_spot_app/01_value_strategy/02_Product_NorthStar.md)**: 目指すべき究極のユーザー体験（Zero-friction Experience）。

## 2. コミュニケーションチャネル
* **GitHub Issues / Discussions**: 機能提案、バグ報告、技術的な議論は基本的にここで行います。なるべくGitHub上で完結させてください。
* **Slack**: 非エンジニアのメンバー（企画やドメインエキスパート等）とのやり取り窓口です。（※参加リンクは[こちら](https://team-mirai-volunteer.slack.com/archives/C0ACBTPPWG5)）

## 3. 貢献のワークフロー

1. **Issueを作成する**: 新しい機能やバグ修正に取り組む前に、必ずIssueを立てて目的を共有してください。
2. **ミッションオーナーのレビュー**: 提案内容がプロジェクトのミッションに合致するか、審査を行います。ミッションから逸脱する機能（Feature Creep）は却下されることがありますが、これは誰もが迷いなく価値創造に集中するためのプロセスです。
3. **実装とPR作成**: 実装を開始し、Pull Requestを作成します。

開発における具体的なルールや環境構築については、以下のドキュメントを参照してください。

* 💻 **[開発環境セットアップガイド (setup.md)](./setup.md)**
* 🔄 **[開発ワークフローとルール (workflow.md)](./workflow.md)**
