# ExternalApi（外部API提供）

## 概要
演説スポットBaseのデータを外部アプリケーションに読み取り専用で提供するAPI。

## 対象ユーザー
- **API Consumer（外部アプリ開発者）**: データ取得
- **API Admin（Mission Owner/Dev）**: キー管理

## 方針・制約
- **読み取り専用:** 書き込み・更新・削除は一切許可しない。
- **レート制限:** APIキーごとに1日最大10回。
- **データコピー前提:** 外部アプリは自身のDBにデータをコピーして利用する。
- **サーバー間通信のみ:** ブラウザからの直接アクセスは想定しない。
- **レスポンスキャッシュなし:** 各APIコールで最新データを返却。
- **ライセンス:** AGPL-3.0。

## スポットデータ取得

### エンドポイント
`GET /api/v1/spots`

### 認証
`X-API-Key` ヘッダー必須。

### レスポンス
GeoJSON形式（`FeatureCollection`）で全スポットデータを一括返却。

レスポンスに含まれる情報：
- メタデータ（帰属表示、ライセンス情報、生成日時、総件数）
- 各スポットの位置情報（GeoJSON準拠: `coordinates` は `[経度, 緯度]` の順）
- 各スポットの属性情報（タイトル、説明、推奨Lv、時間帯、聴衆属性、選挙カー可否、画像URL、日時）

### エラー

| HTTP Status | 説明 |
|:---|:---|
| 401 | APIキーが無効または未指定 |
| 403 | APIキーが無効化されている |
| 429 | 1日のアクセス上限（10回）超過 |
| 500 | サーバー内部エラー |

## APIキー管理

### アクセス
URL: `/admin/api-keys`。API管理専用パスコードで認証。

### 機能
- **キー発行:** アプリ名を入力してAPIキーを生成。生成されたキーは発行時に1回だけ表示される。キーの形式は `mssb_` + ランダム文字列。
- **キー管理:** 発行済みキーの一覧表示（プレフィックス `mssb_` + 先頭8文字のみ表示）、有効/無効の切り替え。
