# セキュリティポリシー (Security Policy)

## サポートされているバージョン
本プロジェクトは、常に `main` ブランチの最新バージョンをサポート対象とします。

| バージョン | サポート状況 |
| :------- | :-------- |
| `main` ブランチ | サポートあり |
| 古いコミット/フォーク | サポートなし |

## 脆弱性の報告方法

本プロジェクト（みらい演説スポットBase）において発見されたセキュリティ上の脆弱性は、通常の公開Issueとして**報告しないでください**。悪意ある第三者への情報漏洩を防ぐためです。

脆弱性の報告は、GitHubが提供する「プライベート報告機能（Private vulnerability reporting）」を活用してください。これにより、公開されることなく安全に開発チーム（メンテナー）へと通知され、GitHub上でセキュアにやり取りおよび管理を行うことができます。

**【報告手順】**
1. 本リポジトリの **[Security]** タブを開きます。
2. 左側メニューの **[Advisories]** を選択します。
3. **[Report a vulnerability]** ボタンをクリックし、詳細を入力して送信してください。

報告を受領した後、チームにて速やかに確認とクローズドな環境での修正（Private fork等を利用）を行い、修正が完了・リリースされ次第、報告者へ進捗をご連絡いたします。

## ご協力のお願い
* 報告前に、本リポジトリおよびシステムの脆弱性であることをご確認ください。
* 脆弱性を悪用したテストや、サービスに影響を与えるような攻撃（DoS等）はご遠慮ください。
* プロジェクト側で修正を適用・公開するまでは、脆弱性に関する情報の外部公開をお控えいただきますようお願いいたします。
